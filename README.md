# 🔎 SOC Incident Investigation: Splunk Log Analysis

## 📌 Overview
This repository documents a Security Operations Center (SOC) incident response investigation. The primary objective of this task was to utilize **Splunk SIEM** to ingest, parse, and analyze sample server logs to investigate a series of suspicious failed login attempts, indicative of a brute-force or credential stuffing attack.

## 🎯 Investigation Objectives
*   **Log Ingestion & Parsing:** Configure Splunk to properly ingest the provided sample logs.
*   **Threat Hunting:** Write effective Splunk Search Processing Language (SPL) queries to filter and identify failed authentication events.
*   **Data Extraction:** Identify the targeted accounts, the frequency of the attacks, and the originating IP addresses of the threat actor.
*   **Incident Reporting:** Compile the findings into a professional SOC incident report with actionable mitigation strategies.

## 🛠️ Tools & Technologies Used
*   **Splunk Enterprise** (SIEM)
*   **Splunk SPL** (Search Processing Language)
*   **Log Analysis & Event Correlation**
